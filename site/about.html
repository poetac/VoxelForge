An honest tool for an unforgiving problem.

voxelforge generates regen-cooled liquid rocket engines as printable STL files. It does not promise rocket science is easy. It promises the hand-cranked early arithmetic is.

What it is

voxelforge takes three numbers — thrust, chamber pressure, propellant pair — and returns a buildable thrust chamber. Contour, regen channels, injector face, mounting flanges, sensor bosses. One STL, ready for an LPBF print bed.

Behind that small input surface is a 34-dimension simulated-annealing search space and a ring of 65 rocket feasibility gates (196 project-wide across 5 physics pillars — 65 rocket · 40 air-breathing · 54 electric · 22 marine · 15 nuclear) that reject any candidate violating rotordynamic, cavitation, body-interference, combustion-stability, LPBF-printability, hull-buckling, or numerical-stability constraints — before any expensive voxel work runs.

Most preliminary-design tools optimise first and validate second. voxelforge inverts that: physics reality is an admission criterion, not a penalty term.

Why voxel-implicit geometry

Mesh-based CAD struggles with the geometry additive manufacturing actually unlocks. Conformal TPMS coolant channels. Monolithic fusion of chamber, turbopump, and feed manifold into a single part. Aerospike plug cooling with internally-routed regen passages.

voxelforge runs on PicoGK, a voxel-implicit kernel from LEAP 71, where those operations are boolean-trivial. A coolant channel isn't a swept surface — it's the negative of a Schwarz P field clipped to a chamber shell. Subtract, add, smooth.

How it differs

Mesh CAD (SolidWorks · Fusion)

Parametric features, history tree. Excellent for machined parts. Struggles when 80 helical channels need to merge with a manifold and clear a turbopump volute in the same body.

voxelforge

Implicit SDF, boolean composition. Subsystems fuse into one part. The only cost is voxel resolution; geometry complexity is free.

Hand calculation (Sutton · Huzel & Huang)

Definitive textbook arithmetic. Slow when you want to sweep 34 dimensions and check up to 196 gates per candidate.

voxelforge

Same correlations, automated. Bartz, Dittus-Boelter, Crocco N-τ, Rao bell — all cited, all bounded by uncertainty bands you can read.

Who it's for

Aerospace and propulsion engineers in the preliminary-design phase. Hobbyist rocketry teams whose ambitions outpace their CAD time. Researchers who need a citable reference design they can perturb and rerun.

Not for: production hardware that hasn't been through CFD, FEA, and a real test stand. See below.

What it doesn't do

Passing every gate means "this design is not obviously broken in the ways we know how to check." It does not mean "this design will fly." voxelforge is a screen, not a certification.

  • No 3D CFD. The Bartz correlation has a ±25-50 % uncertainty band — fine for screening, not for first-fire.
  • No 3D FEA. Pressure-vessel + thermal-stress is axisymmetric; ±15 % on peak stress.
  • No transient acoustics. Crocco N-τ catches chug, buzz, screech as classes; full thermoacoustic modelling is future work.
  • No vendor toolpath. The LPBF printability gates (overhang, trapped-powder, drain-path) shipped in Sprint 27 cover DfM screening; actual slicer integration belongs to the LPBF vendor's API.

The full honest list lives in Voxelforge/docs/LIMITATIONS.md.

Provenance

Every shipped sprint freezes a JSONL fingerprint baseline. The post-cascade diff is the cascade's measured impact. Every example reproduces bit-identically from a SHA-256 receipt. Every gate has a stable machine-readable ID — changing it is a breaking change.

This is not a research toy. It's a tracked, tested, audit-grade tool with 5,700+ passing tests and zero TODO/FIXME/HACK markers in the source tree.

External integration

voxelforge-eval (shipped 2026-04-25, S 38a · T2.2) — a subprocess oracle that wraps the headless .Core physics. Pipe a JSON-line design vector to stdin; get a feasibility report and headline metrics back on stdout. External Python / Julia / R drivers can run the optimisation loop without the .NET runtime — useful for grafting voxelforge onto a Bayesian-optimisation framework, an SU2 closed loop, or a Jupyter exploration session.

Sobol sensitivity indices (shipped 2026-04-25, S 38a · OOB-5) — Saltelli sampling exposes first-order and total-order indices for every design variable against every objective. The first sensitivity heatmap that doesn't require running an external R / Python pipeline.

Multi-chain parallel SA (shipped 2026-04-25, ADR-017) — N independent SA chains in lockstep with elite migration; per-chain Sobol slice for low-discrepancy initial coverage. Auto-scales to ProcessorCount − 2. Strict determinism: same seed + chain count → identical result.
