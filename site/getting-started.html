Get started

From zero to first STL in fifteen minutes.

voxelforge is a .NET 9 / Windows project. PicoGK requires Windows for the OpenVDB build path. Linux and macOS work for headless physics — STL export needs Windows or a Win10 VM.

Prerequisites

OS
Windows 10/11 · x64
RAM
64 GB (ADR-006)
.NET
9.0 SDK
PicoGK
2.0.0 (pinned)
Disk
~6 GB build + voxel scratch
GPU
not required
01

Clone and restore

The full repo is ~280 MB; voxel scratch space is created on first run, not committed.

# clone
git clone https://github.com/poetac/voxelforge.git
cd voxelforge

# restore
dotnet restore voxelforge.sln
02

Run the test suite

5,700+ tests should pass (~17 skipped). Tests do not touch .Voxels directly — they invoke the StlExporter as a subprocess. A clean run is the project's heartbeat.

dotnet test Voxelforge.Tests/Voxelforge.Tests.csproj --logger "console;verbosity=normal"

# expected: 5,700+ passed, ~17 skipped, 0 failed
03

Generate the 5 kN reference design

The canonical example. Inputs are baked into the example folder; output STL goes to out/.

dotnet run --project Voxelforge/Voxelforge.csproj -- \
    --example small-pf-5kn \
    --out ./out/

# first run: ~14 min
# subsequent runs hit the SA cache: ~45 s
If receipt.json in out/ matches examples/small-pf-5kn/receipt.json, you have a verified reproducible build. If it doesn't, that's a bug — open an issue.
04

Inspect the gates report

Every run writes a gates.jsonl with one line per gate, marked pass/fail/margin. Use it to see why the SA accepted or rejected a candidate.

# pretty-print failed-gate summary
jq 'select(.passed == false) | {gate: .id, margin: .margin, value: .value}' \
    out/gates.jsonl
05

Author your own design

Three numbers — thrust, chamber pressure, propellant pair — get you a starting point. Everything else has a sensible default the SA can perturb.

# Program.cs · minimal driver
var spec = new EngineSpec {
    Thrust       = 12_000,        // N (sea-level)
    ChamberP     = 6_000_000,     // Pa
    Propellants  = PropellantPair.LOX_CH4,
    Cycle        = CycleType.GasGenerator,
};

var result = await Forge.Run(spec, opts: new ForgeOptions {
    SAEvalBudget   = 6000,
    OutputDir      = "./out",
    EmitReceipt    = true,
});

Console.WriteLine($"Mass: {result.MassKg:F2} kg");
Console.WriteLine($"ISP_vac: {result.IspVac:F1} s");
Console.WriteLine($"STL: {result.StlPath}");
06

Evaluate designs in bulk (optional)

voxelforge-eval is a subprocess oracle — it wraps the headless .Core physics behind a JSON-line stdin/stdout protocol. Pipe a design in, get a score + feasibility report back. Useful for driving optimisation from Python, Julia, or R without touching the .NET runtime.

# one-shot: pipe a design JSON, receive score + gate report on stdout
echo '{"Conditions":{"Thrust_N":15000,"ChamberPressure_Pa":4000000,
  "MixtureRatio":3.3,"PropellantPair":"LOX_CH4","EngineCycle":"GasGenerator",
  "CoolantInletTemp_K":150,"CoolantInletPressure_Pa":12000000},
  "Design":{"ContractionRatio":6,"ExpansionRatio":16,"ChannelCount":80,
  "ChannelTopology":"Axial"}}' \
  | dotnet run --project Voxelforge.Eval/Voxelforge.Eval.csproj

# streaming JSONL mode — one design object per line, EOF to exit
dotnet run --project Voxelforge.Eval/Voxelforge.Eval.csproj -- --jsonl < designs.jsonl

Output is {"id":null,"score":{...}}; infeasible designs return "TotalScore":+Infinity. The full OperatingConditions and RegenChamberDesign schemas match the C# record definitions in Voxelforge.Core.

07

Iterate with the GUI (optional)

The Voxelforge WinForms app exposes the same engine spec as a parameter panel and renders the STL inline via the embedded PicoGK viewer. Useful for parameter exploration; the headless path is canonical.

08

Print it

The output STL is sized for a 250 × 250 × 280 mm LPBF envelope by default. voxelforge does not generate supports or slicer instructions — that lives in your printer's tooling. The printability.json sidecar lists overhang regions and recommended build orientation.

Real talk. Passing every gate means voxelforge believes the design is geometrically and thermodynamically self-consistent. It does not mean it will fire safely. CFD, FEA, hot-fire on a test stand. In that order.

Next steps

  • Open an issue if a correlation, gate, or builder is missing — the project is actively accepting contributions.
