Architecture

Why split

Core has zero kernel dependencies.

The Sprint-A1/A2 split landed in 2026-04-25 (ADR-015 + ADR-016). 117 .cs files moved into .Core; 19 voxel builders into .Voxels. Result: an SU2 subprocess oracle, a Python BoTorch driver, a CFD validation closed loop — all become straightforward consumers of Voxelforge.Core.dll.

Two pitfalls (hard-won)

xUnit + PicoGK do not co-exist.

Instantiating new PicoGK.Library(vox) inside an xUnit test crashes the host on dispose (GLFW / OpenVDB teardown). Voxel-building tests live in .Benchmarks or .StlExporter; tests invoke them via Process.Start and parse stdout. This is structural — .Tests does not directly reference .Voxels.
